# バット工房 iOS — 仕様書

本書は `~/Desktop/Coding/batios`（`bat` から複製したiOSアプリ版）の**製品仕様**を定義する。
方針は確定済み:

- **アーキテクチャ**: 案A = Capacitor継続 + Three.js(r128)描画強化
- **グラフィック**: 現行のネオン・ナイター様式を**強化**（フルリニューアルや写実化はしない）
- **ランキング**: **Game Center**（アプリ専用。Web版のGAS共有ランキングとは別系統・混ぜない）
- **配布形態**: batios は**ローカルiOS開発専用**。GitHub連携なし（gitリモートは外す）。
- **死守ライン**: 物理・ゲームバランス（飛距離の指数関数、スイング補正、球速カーブ、
  バット計測値）は現行数値を変更しない。改修は描画・UI・ネイティブ統合に閉じる。

> 開発の手順・タイムラインは Web版リポジトリの `docs/ios-app-plan.md` 参照。本書は「何を作るか」。

## 1. プロダクト概要

- **ジャンル**: 3D野球・バット製作 + 打撃チャレンジ
- **プラットフォーム**: iOS（iPhone・縦画面固定）。iPadは将来対応（横レイアウト別途）。
- **プレイ単位**: 1セッション = バットを削る → 10球打つ → 総飛距離をランキング登録
- **オフライン**: 完全動作（Three.jsローカル化済み）。ランキング送信のみ通信。
- **対象**: カジュアル。ログイン不要。

## 2. ゲームフロー仕様（現行踏襲・変更不可）

```
[工房モード] ──完成──▶ [打席モード(10球)] ──10球終了──▶ [結果/ランキング]
     ▲                                                         │
     └──────────────── 工房に戻る ◀───────────────────────────┘
```

### 2-1. 工房モード（クラフト）
- 横向きに回転する丸太を鉋（カーソル/タッチ追従）で削ってバット形状を作る。
- バットの素材を6種から選択: **木製・カーボン・アルミ・金・ダイヤ・黒曜石**。
- リアルタイムで計測値を表示: 質量 / 重心 / スイング時間 / 芯の直径 / レーダー（パワー・操作性・しなり）。
- 削り形状 `radii[]`（56セグメント）から質量・重心・慣性モーメント・COPを実計算。

### 2-2. 打席モード
- 投球は10球。球速は `62 + 球数*4.8 + 乱数*4` km/h（1球目≈67 → 10球目≈110）。
- 投球前に2秒のワインドアップゲージ（円が収縮）→ 0で発射。
- 投球位置・ミート位置は固定（モバイル対応で確定済み。マウス追従ミートは廃止済み）。
- タップ/クリック/スペースでスイング。長押しドラッグで視点。
- 接触時にバット面法線で反射、しなり・体重移動補正(BAT_BOOST=1.9)を加味。
- 飛距離は指数関数 `mult = exp((E-0.82)*3.2)`（上限12）。HUDに打球速度・角度・方向・飛距離。
- 打球にカメラ追従。場外・宇宙到達などの演出あり。

### 2-3. 結果・ランキング
- 10球の総飛距離を表示。
- **Game Centerリーダーボード**に総飛距離をスコア送信（§3）。名前はGame Centerのニックネーム。
- アプリ内のランキング表示はローカル保存（直近記録）を出しつつ、Game Centerのランキングへ誘導。
- 使用素材はローカル記録に併記（Game Centerはスコア値のみ。素材はローカル表示で見せる）。

## 3. グラフィック強化仕様（本バージョンの主目的）

現行のネオン・ナイター様式を保ったまま、以下を追加・強化する。**ジオメトリの当たり判定
座標・バットLathe形状・カメラ基準位置は変更しない**（描画属性のみ変更）。

### 3-1. ポストプロセス（最優先）
- `EffectComposer` + `UnrealBloomPass` を導入。
- 発光対象: 照明塔、スコアボード、月・星、打球のハロー、ジャストミート演出。
- パラメータ初期値（実機調整前提）: strength 0.8 / radius 0.4 / threshold 0.85。
- アンチエイリアス: `renderer` のMSAA or `SMAAPass`。
- **負荷対策**: `setPixelRatio(min(devicePixelRatio, 2))`。品質トグルでBloom無効化可能に（§5）。

### 3-2. マテリアルのPBR化
- 主要オブジェクトを `MeshStandardMaterial` 化（metalness/roughness）。
- バット6素材の質感差を強化（色だけでなく反射特性で差別化）:
  | 素材 | metalness | roughness | 備考 |
  |---|---|---|---|
  | 木製 | 0.0 | 0.7 | 木目 |
  | カーボン | 0.1 | 0.3 | 綾織りノーマル |
  | アルミ | 0.9 | 0.25 | 環境反射 |
  | 金 | 1.0 | 0.15 | 強反射・ネオン映え |
  | ダイヤ | 0.2 | 0.0 | 透過+envMap |
  | 黒曜石 | 0.6 | 0.05 | 黒く磨いた反射 |
- **環境マップ1枚**（夜空キューブマップ）を共有して金属・ガラス系を底上げ。

### 3-3. 打撃エフェクト
- ジャストミート: 火花パーティクル + フラッシュ + 軽いカメラシェイク。
- 打球トレイル: 現行の球列 → ライン+Bloomで光の軌跡。
- 場外/宇宙: 衝撃波リングスプライトで到達演出を強化。

### 3-4. 環境ディテール（任意・余力で）
- 芝の刈り分けストライプ、土のラフネス、観客の密度感向上。
- ライト塔からの指向性ライト + 選択的シャドウ（バット/ボール/打者のみ）。

### 3-5. UIトーン
- グラスモーフィズム・余白・角丸を統一。モード遷移にトランジション。
- セーフエリア対応は実装済み。アイコン刷新。

## 4. iOSネイティブ統合仕様

| 機能 | 実装 | 目的 |
|---|---|---|
| **Game Center ランキング** | リーダーボード「総飛距離」にスコア送信（下記詳細） | 公式公開ランキング |
| ハプティクス | `@capacitor/haptics`。ジャストミート=Heavy、ヒット=Medium | 打撃の触感。アプリ化の核 |
| 消音スイッチ対策 | `AppDelegate.swift` で `AVAudioSession` を `.playback` | サイレント時も効果音を鳴らす |
| 画面向き | 縦固定（Info.plist設定済み） | レイアウト保証 |
| ステータスバー | 非表示（設定済み） | 没入感 |
| 起動画面/アイコン | 新トーンで作成 | ストア品質 |

### 4-1. Game Center 仕様（採用確定）
- **前提**: Apple Developer Program 登録が必須（年99 USD）。登録後に App Store Connect で設定。
- **リーダーボード**: 1つ作成。
  - 識別子: `batcraft_total_distance`（仮）
  - スコア形式: 整数（メートル）/ 高い方が上位 / 単位サフィックス `m`
  - 集計: 累積ではなく「ベストスコア」（その人の最高総飛距離が残る）
- **Capacitorプラグイン**: `@openforge/capacitor-game-connect`（または `capacitor-game-services`）を採用。
  - 起動時に `signIn()` でGame Center認証（失敗してもゲームは続行＝ローカル記録のみ）。
  - 結果画面で `submitScore({ leaderboardId, totalScore })`。
  - 「ランキングを見る」ボタンで `showLeaderboard()` ネイティブUIを開く。
- **素材の扱い**: Game Centerはスコア値のみ。使用素材はアプリ内ローカル記録に併記して表示。
- **フォールバック**: Game Center未認証・オフライン時はローカル保存ランキング（現行ロジック）で動作。
- **コード状態**: 現在 `RANK_API=''`（GAS共有は無効化済み）。Game Center連携は本フェーズで追加実装。

## 5. 設定・品質トグル（新規）

- **描画品質 高/低** トグルを設定に追加:
  - 高: Bloom ON / 影 ON / pixelRatio最大2
  - 低: Bloom OFF / 影 OFF / pixelRatio 1.5
- 低スペック端末・発熱時のフォールバック。デフォルトは端末性能で自動判定 or 高。
